Rozszerzenie: Xray Test Generator
Ostatnia aktualizacja: 13 marca 2026

1. Kto jest administratorem danych

Rozszerzenie jest narzędziem deweloperskim działającym wyłącznie po stronie klienta. Deweloper rozszerzenia nie gromadzi, nie przetwarza ani nie ma dostępu do żadnych danych użytkownika. Wszystkie dane pozostają na urządzeniu użytkownika lub są przekazywane bezpośrednio do usług konfigurowanych przez samego użytkownika.

2. Jakie dane są przetwarzane

Rozszerzenie przetwarza następujące dane podane dobrowolnie przez użytkownika:

  • Adres URL instancji Jira (np. https://firma.atlassian.net)
  • Adres e-mail konta Jira
  • Klucz API Jira — token uwierzytelniający do Jira Cloud
  • Klucz projektu, komponent, wersja — parametry konfiguracyjne
  • Zapytanie JQL — filtr wyboru zadań
  • Xray Client ID / Client Secret (opcjonalnie) — dane do Xray Cloud
  • Klucz API Google Gemini (opcjonalnie) — do generowania kroków AI

3. Gdzie dane są przechowywane

Wszystkie powyższe dane są przechowywane wyłącznie lokalnie w przeglądarce użytkownika przy użyciu chrome.storage.local. Deweloper rozszerzenia nie ma do nich żadnego dostępu.

4. Do jakich zewnętrznych usług dane są wysyłane

Dane są przesyłane wyłącznie do usług wskazanych przez użytkownika w konfiguracji:

  • Jira Cloud (Atlassian) — adres e-mail i klucz API służą do autoryzacji żądań REST API (tworzenie zadań, wyszukiwanie, linkowanie). Dane trafiają wyłącznie na instancję Jira wskazaną przez użytkownika.
  • Xray Cloud (eu.xray.cloud.getxray.app) — opcjonalnie, gdy użytkownik włączy generowanie kroków AI. Client ID i Client Secret służą wyłącznie do pobrania tokenu JWT i wykonania mutacji GraphQL dodających kroki testowe.
  • Google Gemini API (generativelanguage.googleapis.com) — opcjonalnie, gdy użytkownik włączy generowanie kroków AI. Przesyłany jest tytuł i opis zadania Jira w celu wygenerowania kroków testowych. Obowiązuje Polityka prywatności Google.

Żadne dane nie są wysyłane na serwery dewelopera rozszerzenia ani żadne inne serwery poza wymienionymi powyżej.

5. Cel przetwarzania danych

Dane są przetwarzane wyłącznie w celu realizacji funkcji rozszerzenia: automatycznego tworzenia artefaktów testowych Xray (Test Cases, Test Plans, Test Executions) w Jira Cloud na podstawie zapytania JQL podanego przez użytkownika.

6. Bezpieczeństwo

  • Wszystkie połączenia z zewnętrznymi API odbywają się wyłącznie przez HTTPS.
  • Klucze API i dane uwierzytelniające są przechowywane lokalnie w przeglądarce i nigdy nie opuszczają urządzenia użytkownika w żaden inny sposób niż opisany w sekcji 4.
  • Rozszerzenie nie wykonuje żadnego zdalnego kodu — cały kod jest zawarty w pliku rozszerzenia.

7. Prawa użytkownika

Użytkownik ma pełną kontrolę nad swoimi danymi:

  • Dane konfiguracyjne można usunąć w dowolnym momencie przez odinstalowanie rozszerzenia lub wyczyszczenie danych rozszerzenia w ustawieniach Chrome (chrome://extensions/).
  • Funkcje AI (Gemini, Xray Cloud) są całkowicie opcjonalne — klucze nie muszą być podawane, jeśli użytkownik nie korzysta z tej funkcji.

8. Zmiany polityki prywatności

Wszelkie zmiany tej polityki będą publikowane na tej stronie z aktualizowaną datą. Dalsze korzystanie z rozszerzenia po wprowadzeniu zmian oznacza ich akceptację.

9. Kontakt

W razie pytań dotyczących prywatności skontaktuj się przez sekcję Issues w repozytorium GitHub rozszerzenia.

Privacy Policy

Extension: Xray Test Generator
Last updated: March 13, 2026

1. Data Controller

This extension is a client-side developer tool. The extension developer does not collect, process, or have access to any user data. All data remains on the user's device or is transmitted directly to services configured by the user.

2. Data Processed

The extension processes the following data voluntarily provided by the user:

  • Jira instance URL (e.g. https://company.atlassian.net)
  • Jira account email address
  • Jira API Key — authentication token for Jira Cloud
  • Project key, component name, fix version — configuration parameters
  • JQL query — issue selection filter
  • Xray Client ID / Client Secret (optional) — credentials for Xray Cloud
  • Google Gemini API Key (optional) — for AI step generation

3. Data Storage

All of the above data is stored exclusively locally in the user's browser using chrome.storage.local. The extension developer has no access to this data.

4. External Services

Data is transmitted only to services designated by the user in the configuration:

  • Jira Cloud (Atlassian) — email and API key are used to authorize REST API requests (creating issues, searching, linking). Data is sent only to the Jira instance specified by the user.
  • Xray Cloud (eu.xray.cloud.getxray.app) — optionally, when the user enables AI step generation. Client ID and Client Secret are used solely to obtain a JWT token and execute GraphQL mutations to add test steps.
  • Google Gemini API (generativelanguage.googleapis.com) — optionally, when the user enables AI step generation. The Jira issue title and description are sent to generate test steps. Google's Privacy Policy applies.

No data is sent to the extension developer's servers or any other servers beyond those listed above.

5. Purpose of Processing

Data is processed solely to provide the extension's functionality: automated creation of Xray test artifacts (Test Cases, Test Plans, Test Executions) in Jira Cloud based on a JQL query provided by the user.

6. Security

  • All connections to external APIs use HTTPS only.
  • API keys and credentials are stored locally in the browser and never leave the user's device in any way other than described in section 4.
  • The extension does not execute any remote code — all code is bundled within the extension package.

7. User Rights

Users have full control over their data:

  • Configuration data can be deleted at any time by uninstalling the extension or clearing extension data in Chrome settings (chrome://extensions/).
  • AI features (Gemini, Xray Cloud) are entirely optional — API keys do not need to be provided if the user does not use these features.

8. Changes to This Policy

Any changes to this policy will be published on this page with an updated date. Continued use of the extension after changes constitutes acceptance.

9. Contact

For privacy-related questions, please contact via the Issues section of the extension's GitHub repository.
